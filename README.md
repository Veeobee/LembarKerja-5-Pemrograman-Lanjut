# LembarKerja-5-Pemrograman-Lanjut
Pada sistem ini, terdapat tiga interface utama yaitu MedicalRecord, Versioned, dan Confidential yang berperan sebagai aturan dasar bagi setiap class data pasien. MedicalRecord memastikan setiap data memiliki identitas unik melalui patientId, sedangkan Versioned digunakan untuk menandai versi data agar sistem dapat membedakan antara format lama dan baru. Sementara itu, Confidential mengatur aspek keamanan data, seperti menentukan tingkat keamanan dan bagaimana data ditampilkan sesuai dengan level akses pengguna. Dengan adanya ketiga interface ini, semua class yang mengelola data pasien memiliki struktur yang seragam sehingga dapat diproses dengan cara yang sama oleh sistem.

Class PatientProfileV1 merupakan representasi data pasien versi awal yang masih sederhana, hanya mencakup patientId, name, dan ssn. Walaupun sederhana, class ini tetap mengikuti semua aturan dari interface yang ada. Class ini juga memiliki mekanisme masking data, di mana informasi sensitif seperti SSN akan disembunyikan jika level akses pengguna rendah, dan hanya ditampilkan jika level akses mencukupi. Class ini menjadi salah satu contoh data yang nantinya akan diproses oleh sistem utama.

Selanjutnya, PatientProfileV2 adalah pengembangan dari versi sebelumnya dengan data yang lebih lengkap, seperti riwayat alergi dan riwayat medis. Karena datanya lebih sensitif, tingkat keamanannya juga lebih tinggi. Proses masking pada class ini lebih detail, di mana semakin tinggi level akses pengguna, semakin banyak data yang dapat dilihat. Hal ini menunjukkan bagaimana sistem dapat berkembang tanpa mengubah struktur dasar, karena tetap mengikuti interface yang sama.

Class SecureResponse digunakan sebagai pembungkus hasil yang diberikan oleh sistem. Class ini menyimpan data yang akan ditampilkan serta informasi apakah data tersebut sudah melalui proses masking atau belum. Dengan pemisahan ini, sistem lain dapat dengan mudah mengetahui status keamanan dari data yang diterima tanpa harus memeriksa isinya secara langsung.

Bagian inti dari sistem terletak pada class IntegrationGateway, yang bertugas mengatur proses pengambilan dan penyajian data pasien. Class ini hanya dapat memproses objek yang memenuhi ketiga interface sekaligus, sehingga menjamin konsistensi data. Alurnya dimulai dengan pengecekan patientId, lalu dilanjutkan dengan membandingkan level akses pengguna dengan tingkat keamanan data. Jika akses tidak mencukupi, sistem akan menampilkan data yang sudah dimasking, sedangkan jika akses cukup, data asli akan ditampilkan. Karena setiap class sudah memiliki cara masking sendiri, gateway tidak perlu membedakan tipe objek secara manual, sehingga kode menjadi lebih rapi dan fleksibel.

Terakhir, MainSimulation berfungsi sebagai program uji coba untuk melihat bagaimana seluruh sistem bekerja. Pada bagian ini dibuat beberapa objek dari versi data yang berbeda, lalu diuji dengan berbagai level akses. Dari simulasi ini dapat dilihat bagaimana data diproses, mulai dari input, pengecekan keamanan, hingga hasil akhir yang dikembalikan dalam bentuk SecureResponse. Hal ini menunjukkan bahwa sistem mampu menangani berbagai versi data dengan cara yang konsisten tanpa perlu mengubah logika utama.
